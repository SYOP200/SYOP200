## 🔧 Setup Instructions (read before publishing)

1. **Create the special repo**: make a new public repo named exactly `SYOP200` (same as your username) — GitHub turns this into your profile README automatically. Put this file in it as `README.md`.
2. **Stats/trophies/streak images** work immediately — no setup needed, they're live services keyed to your username.
3. **Snake animation** needs a one-time GitHub Action:
   - In your `SYOP200` repo, go to **Settings → Actions → General** and enable "Read and write permissions" under Workflow permissions.
   - Create `.github/workflows/snake.yml` with:
```yaml
     name: Generate Snake
     on:
       schedule:
         - cron: "0 */6 * * *"
       workflow_dispatch: {}
       push:
         branches: [ main ]
     jobs:
       generate:
         runs-on: ubuntu-latest
         steps:
           - uses: Platane/snk/svg-only@v3
             with:
               github_user_name: SYOP200
               outputs: |
                 dist/github-contribution-grid-snake-dark.svg
           - uses: actions/checkout@v4
             with:
               ref: output
           - uses: crazy-max/ghaction-github-pages@v4
             with:
               target_branch: output
               build_dir: dist
             env:
               GITHUB_TOKEN: ${{ secrets.GITHUB_TOKEN }}
```
   - Run the workflow once manually (Actions tab → Generate Snake → Run workflow). It creates an `output` branch the README already points to.
4. Swap every placeholder link (LinkedIn, Twitter, portfolio, email) for your real ones.
5. Want a different color? Change the hex codes in the `theme=dark&bg_color=...&title_color=...` query params on any image URL — all these services accept custom colors.

### 🩹 If the Stats/Langs cards or Trophies still don't load

These two widgets are free community-run services (`github-readme-stats.vercel.app` and `github-profile-trophy.vercel.app`). They're shared by thousands of profiles, so they routinely get rate-limited or briefly go down — that's what you're hitting, not a mistake in the README. Two options:

**Option A — just wait / hard-refresh.** Often the image comes back within a few minutes. Right-click → "Open image in new tab" to bypass GitHub's cache and see the live status.

**Option B — self-host your own copy (permanent fix, ~2 min, free):**

- **Stats cards:** go to https://github.com/anuraghazra/github-readme-stats, click **"Deploy to Vercel"** in the README, sign in with GitHub, deploy. Replace `github-readme-stats.vercel.app` in both image URLs above with the new domain Vercel gives you.
- **Trophies:** go to https://github.com/ryo-ma/github-profile-trophy, same process, replace `github-profile-trophy.vercel.app` with your new domain.

Once self-hosted, the images are yours alone and won't be affected by other users' traffic.
